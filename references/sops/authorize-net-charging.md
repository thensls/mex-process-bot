# How/When to Charge a Card in Authorize.net SOP

**Category:** Payments / General

## When to Use This

Manual charges through Authorize.net are not standard — only use in these situations:

- Manually enrolling a member due to a technical issue
- Charging $10 for an additional certificate
- Charging the $5.99 IKF (induction kit fee) not previously added
- Charging for an enrollment plaque
- Charging for a shipping upgrade (Graduation Set early shipping)
- Manually enrolling a member into membership or A&E Certification/Credit Recommended

⚠️ **Always check with SOS first if unsure whether a charge is appropriate.**  
⚠️ **Never charge a member without their written consent.**

---

## Step-by-Step Process

### Step 1: Log into Authorize.net
Log in with your credentials.

### Step 2: Navigate to Tools
Select **Tools** from the menu.

### Step 3: Enter Payment Information

Confirm auto-selected settings:
- Transaction Type: **Authorize & Capture**
- Payment Method: **Charge a Credit Card**

Enter:
- Card number (no spaces)
- Expiration date (MMYY format)
- CVV (3-digit on back; 4-digit for Amex)
- Amount — **double-check the decimal position before proceeding**

📢 Read the card details and amount back to the member to confirm before submitting.

> If the payment is declined, all information is deleted for security — you must re-enter everything from scratch.

### Step 4: Add Order Description
- Invoice #: Leave blank
- Description: Brief description of what you're charging

### Step 5: Enter Customer Billing Info
- First and Last Name
- Address (leave blank for international)
- Email address
- Click **Submit**

Leave all other sections (Shipping, Additional Info) blank.

### Step 6: Document in Feather (Critical)

Every Authorize charge must be reflected in Feather:

1. Open the member's Feather profile → **Transactions** tab
2. Select **"Add Transaction"**
3. Scroll to the **"CREATE TRANSACTION"** box:
   - Package: Select the appropriate item (Certificate, Enrollment, A&E, etc.)
   - Payment Method: **Authorize (manual)**
4. Press **"ADD PACKAGE TRANSACTION"**
5. Verify the transaction appears on the member's Transactions page

### Post-Procedure: Send the Receipt

1. Search for the transaction in Authorize.net
2. Click **Print** on the Transaction Detail page → Print → Save as PDF
3. Email the PDF receipt to the member
4. Include a brief note confirming the charge and offering support

---

## Key Rules

| Rule | Detail |
|---|---|
| Written consent | Required before every manual charge |
| Declined card | Re-enter all info from scratch — data is wiped for security |
| Refunds | Cannot be processed in Authorize — use Refund Exception SOP or ACH Refunds SOP |
| Feather documentation | Every charge must be logged with "Authorize (manual)" as payment method |
| Decimal accuracy | Double-check before submitting — wrong decimal = wrong charge |
| Unsure if charge is appropriate | Check with SOS first |
